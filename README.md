# Olympians-RAG
Persian Q&amp;A system about Greek gods using LangChain + Cohere
# ⚡ Greek Gods Q&A — پرسش و پاسخ هوشمند درباره خدایان یونان

> یک سیستم پرسش‌و‌پاسخ فارسی درباره خدایان المپ، ساخته‌شده با **LangChain** و مدل **Cohere `command-r`**، با تکنیک *Few-Shot Prompting* روی یک پایگاه‌دانش دست‌ساز.

---

## 🏛️ این پروژه چیست؟

این پروژه یک Q&A Engine است که به سوالاتی درباره ۱۴ خدای المپ (زئوس، هرا، پوسوئیدون، آتنا، آرتمیس، آپولون، آرس، هرمس، دیونیسوس، هفایستوس، آفرودیت، دمتر، هستیا و هادس) به زبان فارسی پاسخ می‌دهد — کوتاه، دقیق، و فقط بر اساس اطلاعاتی که در اختیارش گذاشته شده.

مدل **هیچ‌چیزی از خودش نمی‌سازد**: اگر پاسخ سوالی در دیتای داده‌شده نباشد، صادقانه می‌گوید *"نمی‌دانم"* — یک الگوی کلاسیک Grounded QA / Retrieval-style Prompting برای کاهش Hallucination.

```
🗨️  "خواهر زئوس کیست؟"          →  "هستیا"
🗨️  "هرمس چه چیزی را حمل می‌کند؟" →  "گرز"
🗨️  "اسلحه اصلی زئوس چیست؟"      →  "رعد"
```

## 🧠 معماری و نحوه کار

```
┌─────────────────┐     ┌──────────────────────┐     ┌─────────────────┐
│   gods.json       │ ──▶ │  System Prompt         │ ──▶ │                   │
│  (پایگاه‌دانش)     │     │  + اطلاعات خدایان       │     │   ChatCohere       │
└─────────────────┘     │  + ۱۰ نمونه Few-Shot   │     │  command-r-08-2024 │
                          └──────────────────────┘     │                   │
┌─────────────────┐                                    │                   │
│ questions.csv     │ ─────────────────────────────────▶│                   │
│  (سوالات تست)      │                                    └────────┬────────┘
└─────────────────┘                                             │
                                                                    ▼
                                                          پاسخ ۴ تا ۵ کلمه‌ای
```

| مرحله | توضیح |
|---|---|
| **۱. بارگذاری دانش** | اطلاعات هر خدا (`name`, `description`, `appearance`, `features`) از `gods.json` خوانده و به یک متن یکپارچه تبدیل می‌شود. |
| **۲. ساخت System Prompt** | یک دستورالعمل فارسی به مدل می‌گوید: پاسخ را در ۴ تا ۵ کلمه بده، و اگر نمی‌دانی، نمی‌دانم بگو. |
| **۳. Few-Shot Examples** | ۱۰ نمونه سوال‌و‌جواب واقعی برای کالیبره کردن لحن و طول پاسخ مدل به کار می‌رود. |
| **۴. زنجیره LangChain** | `prompt \| model \| parser` — یک Chain ساده و خوانا با `StrOutputParser`. |
| **۵. اجرای دسته‌ای** | برای هر سطر `questions.csv` یک پاسخ گرفته می‌شود؛ هر ۱۵ سوال، ۶۵ ثانیه مکث برای رعایت Rate Limit کوهیر. |

## 📊 دیتاست

### `data/gods.json`
پایگاه‌دانشی شامل **۱۴ خدای المپ**، هر کدام با چهار فیلد:

| فیلد | محتوا |
|---|---|
| `name` | نام خدا (مثلاً «زئوس») |
| `description` | شرح هویت، نسبت خانوادگی، حوزه قدرت |
| `appearance` | توصیف ظاهری در آیکونوگرافی کلاسیک |
| `features` | ابزار، نماد و حیوانات مقدس مرتبط |

### `data/questions.csv`
**۸۹ سوال** فارسی برای ارزیابی مدل — از روابط خانوادگی («همسر زئوس چه کسی است؟») تا جزئیات نمادین («پرنده مقدس هادس چه حیوانی است؟») و حتی یک تله عمدی (سوال درباره «تایتان» و «دیبا» که در دیتاست وجود ندارند، برای تست توانایی مدل در گفتن «نمی‌دانم»).

## 🚀 راه‌اندازی

### ۱. کلون کردن ریپازیتوری
```bash
git clone https://github.com/USERNAME/greek-gods-qa.git
cd greek-gods-qa
```

### ۲. نصب وابستگی‌ها
```bash
pip install -r requirements.txt
```

### ۳. تنظیم کلید API
```bash
cp .env.example .env
```
سپس فایل `.env` را باز کن و کلید Cohere خودت را قرار بده:
```
COHERE_API_KEY=your_actual_key_here
```

> 🔑 کلید API رایگان را می‌توانی از [dashboard.cohere.com](https://dashboard.cohere.com/) بگیری.

### ۴. اجرای نوت‌بوک
```bash
jupyter notebook Olympians.ipynb
```
